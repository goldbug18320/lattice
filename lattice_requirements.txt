This is a war game simulator project. 
Create a ai enabled drone swarm command control platform Lattice.

Features and capabilities:
1) ability to receive life feed from reconnaissance drones on enemy drones, ships, tanks, missile launchers locations.
2) ability to command a swarm of drones to locate, track and attack enemy drones, ships, tanks, missile lauchers.
3) Define a set of APIs to receive life feed from reconnaissance drones on enemy drones, ships, tanks, missile launchers locations.
4) Define a set of APIs to control a swarm of drones to locate, track and attack enemy drones, ships, tanks, missile lauchers.
5) the platform use LLM to assign tasks to a single drone or a swarm of drones. In case of a swarm of drones, it is expected the drones are AI enabled which can coordinate with each other to locate and attach targets by themselves. The platform does not assign tasks to each drone in the swarm. This helps the success rate in GPS denial battle ground.
6) 3D UI with with real world terrain information (such as Taiwan) to show the locations of our drones and enemy drones, ships, tanks, missile launchers.
7) platform accepts natural language commands via text input.
8) LLM determine the appropriate drones to use, based on the payload, range. LLM also determine the swarm size appropriate for the enemy targets.
9) LLM follow operator's requests on UI, such as "show Taiwan on map", etc.
10) All deployed drones should be constantly moving until their max range is reached. 
11) All deployed drones should have their location, speed, battery status live feed to the Lattice.
12) 
13) LLM should determine classify the enemy assets to 1) high value target, such as ship, missile launchers. 2) medium value target, such as tanks. 3) low value targets. And ask human for permission to attack enemy targets according to value classification.
14) LLM should respond with natural language, such as "what is the current status of MQ9-01", instead of just translating inquiry into commands.
15) when the user click on engage button, the combat swarm should show up on the swarm and drone status panel.
16) idling drones should not show up on on the swarm and drone status panel.
17) System support drag and drop deployment of both friendly assets and enemy assets. The system should persist all friendly and enemy assets locations to a configuration file.
18) use appropriate icons on the map for the combat objects. For example, use drone icon for drones. use ship icon for ships. use tank icon for tanks.
19) when user click on an enemy asset in enemy targets panel, the asset should be highlighted on the map. The map should be moved to show the asset.
20) when user click on an asset in swarm & drone status panel, the asset should be highlighted on the map. The map should be moved to show the asset.
21) soldiers, tanks, missile launchers must be on land. ships must be in the sea. drones can be on land, at sea or in the air. 
22) when the user click on engage button on one enemy asset, LLM will select a combat drone swarm that can reach that enemy target. Inform the user if no combat drone swarm can reach the enemy target. the selected drone unit name is displayed and ask the user for approval. Only the select enemy target will be engaged. Once the user approve the request, the engage button on the target should be disabled and the selected drone swarm is moving off the available drone swarm list. (i.e. one drone swarm cannot be used to attach multiple enemy targets). 
23) when the combat swarm reach the target, the target is destroyed, as well as the combat swarm.
24) when the user click on track button on one enemy asset, LLM will select a reconnaissance drone that can reach that enemy target. Inform the user if no reconnaissance drone can reach the enemy target. the selected drone unit name is displayed and ask the user for approval. Only the select enemy target will be tracked. Once the user approve the request, the trace button on the target should be disabled and the selected reconnaissance drone is moving off the available reconnaissance drone list. (i.e. one drone cannot be used to track multiple enemy targets).


Initialization:
1) system loads friendly and enemy assets locations from a configuration file.
2) No default deployments of friendly and enemy assets by the system.
